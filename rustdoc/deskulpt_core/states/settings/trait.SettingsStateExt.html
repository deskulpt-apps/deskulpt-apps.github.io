Provided Methods§

Source

fn manage_settings(&self)

Initialize state management for the settings.

This will load the settings from the persistence directory and initialize the shortcuts. If any step fails, it will fall back to a state that preserves as much persisted data as possible.

Source

fn get_settings(&self) -> RwLockReadGuard<'_, Settings>

Get an immutable reference to the settings.

The returned reference is behind a lock guard, which should be dropped as soon as possible to minimize critical section.

Source

fn get_settings_mut(&self) -> RwLockWriteGuard<'_, Settings>

Get a mutable reference to the settings.

The returned reference is behind a lock guard, which should be dropped as soon as possible to minimize critical section.

Source

fn apply_settings_patch(&self, patch: SettingsPatch) -> Result<()>

Apply a patch to the settings.

See SettingsPatch for more information of how the patch is applied. The patch application is best-effort: any part of the patch that fails to be applied will be skipped, and the rest will be applied as normal. Errors will be accumulated and returned as a single error at the end if any occurred.
